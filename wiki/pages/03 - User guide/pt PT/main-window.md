**<span style="font-size:120%">Elementos da janela principal do Gramps</span>**  
*ou*  
**<span style="font-size:120%">Um guia visual para o ambiente do Gramps</span>**.

Os ambientes gráficos de utilização (GUIs) são uma forma de organizar a informação no ecrã do computador, fácil de entender e usar porque usa ícones, menus e um rato em vez de apenas texto.

Enquanto um GUI é muito mais intuitivo do que memorizar comandos de teclado, é difícil aprender com a documentação sem saber o nome do recurso a procurar. Há muitos elementos no Gramps que não estão rotulados para reduzir a desordem e maximizar o espaço para exibir informações genealógicas. Há até [opções de preferência](wiki:Gramps_6.0_Wiki_Manual_-_Settings/pt_PT#Environment_Settings) para não mostrar rótulos de texto para os botões (ícones) no painel navegador de categorias.

Assim, esta secção do manual dá os nomes padronizados dos elementos do ecrã do Gramps. Armado com os nomes padrão, pode usar mais eficientemente um motor de pesquisa (como o Google) para encontrar páginas no manual em rede que discutam esse elemento. Como exemplo, para pesquisar páginas do manual do Gramps na frase <a href="https://www.google.com/search?q=modo%20navegador+site%3Agramps-project.org%2Fwiki" class="external text" rel="nofollow">"Modo Navegador"</a>, use os seguintes termos de pesquisa do Google: `"modo navegador" site:gramps-project.org/wiki`

Esta pesquisa em particular encontra até mesmo ilustrações com nomes descritivos da selecção dos modos para a barra lateral do navegador. Seleccionando uma dessas imagens mostra uma página web sobre a ilustração e onde ela é usada na documentação. Essas páginas ilustradas tendem a ter um maior material introdutório.

Se um determinado elemento do ecrã não aparecer entre as imagens abaixo, procure na [galeria de capturas de ecrã do Gramps](wiki:Screenshots). Esta página mostra imagens reduzidas do Gramps que são utilizadas em toda a documentação. Clique na miniatura para mostrar a imagem em tamanho real e as variantes dessa imagem, depois vá até a secção "File usage" para encontrar páginas Wiki que ilustrem as características dentro dessa [captura](wiki:Screenshots). Da mesma forma, a [galeria de ícones do Gramps](wiki:Gramps_icon_set) identifica os nomes adequados dos ícones usados em todo o Gramps.

Uma vez familiarizados com os elementos do ecrã, podemos explorar os [atalhos de teclado](wiki:Gramps_6.0_Wiki_Manual_-_Keybindings/pt_PT), criados propositadamente para funcionalidades usadas muito frequentemente.

#### <span id="Menus_pendentes" class="mw-headline">Menus pendentes</span>[edit](wiki:index.php)<span class="mw-editsection-bracket">\]</span>

Além da barra de menus, a ocorrência de um botão não rotulado com uma *seta abaixo* <span class="kbd" style="border: 1px solid; color:#5e4638; background-color: #f9f9f9; border-bottom-width: 2px; -moz-border-radius: 3px; -webkit-border-radius: 3px; border-radius: 3px; letter-spacing:0.1em; padding-left:0.5em; padding-right:0.4em;">∨</span> na <span class="kbd" style="border: 1px solid; color:#5e4638; background-color: #f9f9f9; border-bottom-width: 2px; -moz-border-radius: 3px; -webkit-border-radius: 3px; border-radius: 3px; letter-spacing:0.1em; padding-left:0.5em; padding-right:0.4em;">[barra de menu do gramplet](wiki:Gramps_6.0_Wiki_Manual_-_Main_Window/pt_PT#Gramplet_Bar_Menu)</span> indica um **menu pendente** com opções adicionais disponíveis para o item à sua esquerda.

Qual é a diferença entre um menu [pendente](#Pull-down_menus) e um menu [contextual](#Pop-up_menus)? Ambos partilham uma escolha com o rato. Mas os menus pendentes são revelados com um clique direito num botão ou marcador. Já o menu contextual, é revelado com um clique direito num qualquer elemento do ecrã.

#### <span id="Menus_contextuais" class="mw-headline">Menus contextuais</span>[edit](wiki:index.php)<span class="mw-editsection-bracket">\]</span>

A opção do ambiente mais invisível pode ser o **menu contextual**.

Clicar com o botão direito do rato num item do ecrã revelará um menu de atalhos para operações frequentes sobre esse item (ou registo).

Note que clicar com o botão direito numa linha de uma tabela que não a seleccionada (realçada, ou activa) também revela o menu relacionado com a linha seleccionada. Portanto, clique primeiro com o botão esquerdo no objecto desejado, e só depois com o botão direito.

### <span id="Caixas_de_combinação_com_selector"></span><span id="Caixas_de_combina.C3.A7.C3.A3o_com_selector" class="mw-headline">Caixas de combinação com selector</span>[edit](wiki:index.php)<span class="mw-editsection-bracket">\]</span>

Esta é uma caixa normal de inserção de texto combinada com um botão de lista pendente. O botão permite uma escolha rápida numa lista de tipos de dados pré-definidos, assim como tipos de dados adicionais.

Adicionar um novo [tipo personalizado](wiki:Gramps_Glossary#custom) à lista é simples. Insira o novo termo na caixa de texto. Note que a lista é sensível a maiúsculas. Seja conservador ao criar novos tipos. A categorização torna-se mais confusa e a ordenação menos significativa entre tipos muito semelhantes.

Alguns dos [**tipos *personalizados***](wiki:Gramps_Glossary#custom) que podem ser definidos são: atributos de eventos, atributos de famílias, atributos de multimédia, atributos de indivíduos, [papéis em eventos](wiki:Gramps_6.0_Wiki_Manual_-_Entering_and_editing_data:_detailed_-_part_2/pt_PT#General), [tipos de eventos](wiki:Custom_Event_Types), tipos de parentescos, tipos de referências de filhos, tipos de origem de nomes, tipos de nome, tipos de notas, tipos de local, tipos de repositórios, atributos de fontes, tipos de multimédia de fontes, tipos de URL e tipos de [filtro personalizados](wiki:Gramps_Glossary#filter).

Consolidar ou remover tipos personalizados pode ser complexo. A ferramenta [Limpeza de tipos](wiki:Addon:Types_Cleanup_Tool) limpa registos que utilizem tipos personalizados antigos.

## <span id="Filtros" class="mw-headline">Filtros</span>[edit](wiki:index.php)<span class="mw-editsection-bracket">\]</span>

<div class="thumb tright">

<div class="thumbinner" style="width:402px;">

![[_media/PeopleCategory-sidebar-filter-enabled-60-pt_PT.png]]

As bases de dados genealógicas podem conter informações sobre **muitos** indivíduos, famílias, locais e objectos. Logo, é possível que uma vista contenha uma longa lista de dados com os quais é difícil de trabalhar. Além da básica [procura ao escrever](wiki:Gramps_6.0_Wiki_Manual_-_Navigation/pt_PT#Finding_records), o Gramps oferece dois meios diferentes para filtrar uma lista até obter um tamanho mais "legível".

Estes métodos são a **procura** e a **filtragem**.

Uma procura pesquisará o texto indicado na lista activa, enquanto os filtros mostram indivíduos que cumpram o critério do filtro.

A procura é um método simples mas rápido de pesquisar as colunas mostradas no ecrã. Quando a [barra lateral direita](wiki:Gramps_6.0_Wiki_Manual_-_Main_Window/pt_PT#Bottombar_and_Sidebar) **não** está visível, é mostrada a **<span class="kbd" style="background:#D3D3D3; letter-spacing:0.1em; padding-left:0.5em; padding-right:0.4em;">[barra de procura](wiki:Gramps_6.0_Wiki_Manual_-_Main_Window/pt_PT#Search_Bar)</span>**. Escrever na **<span class="kbd" style="background:#D3D3D3; letter-spacing:0.1em; padding-left:0.5em; padding-right:0.4em;">[barra de procura](wiki:Gramps_6.0_Wiki_Manual_-_Main_Window/pt_PT#Search_Bar)</span>** e clicar em <span class="kbd" style="border: 1px solid; color:#5e4638; background-color: #f9f9f9; border-bottom-width: 2px; -moz-border-radius: 3px; -webkit-border-radius: 3px; border-radius: 3px; letter-spacing:0.1em; padding-left:0.5em; padding-right:0.4em;">Localizar</span> mostrará só as linhas que correspondem ao texto escrito.

Alternativamente, pode activar um filtro na [barra inferior ou na barra lateral](wiki:Gramps_6.0_Wiki_Manual_-_Main_Window/pt_PT#Bottombar_and_Sidebar). Quando a barra lateral de filtro está visível, a **<span class="kbd" style="background:#D3D3D3; letter-spacing:0.1em; padding-left:0.5em; padding-right:0.4em;">barra de procura</span>** está oculta. O filtro permite definir um conjunto de regras a aplicar à lista. Aplica-se baseado nas regras e nos dados, não no que está no ecrã. Também pode construir filtros da categoria mostrada, clicando no correspondente botão do editor no menu **Editar**.

<p>A procura só localiza correspondências exactas do texto. Se a data mostrada for "Jan 1, 2000", uma procura por "1/1/2000" falha, mas um filtro de "1/1/2000" cumpre o critério, porque compara o valor da data, em vez da data formatada.</p></td>
</tr>
</tbody>
</table>

  
Quando o Gramps abre uma árvore genealógica, não há filtros activos. Na categoria Indivíduos, por exemplo, todos os indivíduos na árvore genealógica são listados.

## <span id="Resolução_de_problemas_de_mau_comportamento"></span><span id="Resolu.C3.A7.C3.A3o_de_problemas_de_mau_comportamento" class="mw-headline">Resolução de problemas de mau comportamento</span>[edit](wiki:index.php)<span class="mw-editsection-bracket">\]</span>

Se o GUI do Gramps se está a comportar de forma contrária à descrita neste guia, pode ter um pequeno problema de instalação ou compatibilidade.

É provável que o comportamento já tenha sido visto antes e que uma solução tenha sido identificada.

Consulte as [páginas de resolução de problemas](wiki:Category:Troubleshooting). Se não conseguir descobrir a solução, publique uma descrição do problema na comunidade geral do Gramps, no nosso [fórum dos utilizadores](wiki:Contact#Official). Estamos sempre a ajudar-nos mutuamente.
