## Редактирование информации о местах

Для редактирование информации о местах переключитесь в вид Места и выберите необходимую запись из списка мест. Дважды щелкните эту запись или щелкните кнопку в панели инструментов, чтобы открыть следующий диалог:

### Редактор местоположений

![[_media/PlaceEditor-dialog-example-60.png|Fig. {{#var:chapter}}.{{#vardefineecho:figure|{{#expr:{{#var:figure}}+1}}}} Place editor - dialog]]

В диалоге доступны следующие поля:

1.    
    полное название этого места.

2.    
    положение Места над экватором в виде десятичной дроби или в градусах. Например, правильными будут значения: 12.0154, 50°52'21.92"N, N50º52'21.92" или 50:52:21.92. Вы можете установить эти значения через вид География, найдя в нем Место или через сервис карт в виде Места.

3.    
    положение Места относитально нулевого мередиана (Гринвича) в виде десятичной дроби или в градусах. Например, правильными будут значения: -124.3647, 124°52'21.92"E, E124º52'21.92" или 124:52:21.92. Вы можете установить эти значения через вид География, найдя в нем Место или через сервис карт в виде Места.

4.    
    уникальная запись для идентификации Места, оставьте для заполнения системой.

Вкладка содержит Интернет адреса, имеющие отношение к данному Месту. Внизу находится список таких Интернет адресов, сохраненных в базе данных. Обратите внимание, что кнопки , и становятся доступны, только когда медиа-объект выбран в списке.

#### Ссылки

Вкладка показывает любые записи в базе данных (события и т.п. ), которые ссылаются на это Место. Эта информация не может быть изменена из диалога . Вместо этого, будет открыта соответствующая запись (к примеру, событие рождения) и изменена ссылка на ее Место.

### Note markup and preformat in reports

Markup like **bold**, color, <u>underline</u>, ... can be added to notes. A note can be preformatted or not. It depends on the output type how this markup will appear. Here an overview is given of what you can expect.

- **Pdf** and **direct print** (to printer or to file) fully support the markup and the preformatted setting
- **ascii** print removes all markup from the notes for obvious reasons
- **LaTeX** output interprets the markup to its best of possibilities. LaTeX is not well suited as a typesetting language to add custom style. That would break the benefits LaTex offers. Hence, the following is done:

:\* **bold**, <u>underline</u> and *italic* is supported

:\* fontsize is mapped to the size indicators of LaTeX in a fuzzy manner

:\* mono fonts are shown as a mono spaced font

:\* color and font is not supported

:\* preformatted is handled correctly

- **Narrative Web**. Many people use the Narrative Web report as an easy way to work with their data. This report is trying to respect markup in the notes. This is an interpreted translation, it is not one-to-one.
- **ODF** output does not support markup at the moment.
- **RTF** output does not support markup at the moment.
- **html** output does not support markup at the moment.
